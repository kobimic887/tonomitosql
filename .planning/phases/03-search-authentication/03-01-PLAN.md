    2. **scripts/create-api-key.py** — Create a standalone Python script for generating API keys:

       ```python
       #!/usr/bin/env python3
       """Generate an API key and store its SHA-256 hash in the database.

       Usage:
           python scripts/create-api-key.py [--name KEY_NAME]
           docker compose exec api python scripts/create-api-key.py --name "my-app"

       The script prints the raw API key to stdout (this is the only time it's visible).
       The SHA-256 hash is stored in the api_keys table.
       """
       import argparse
       import hashlib
       import secrets
       import sys

       import psycopg

       from app.config import settings


       def create_api_key(name: str) -> str:
           """Generate a random API key, store its hash, return the raw key."""
           # Generate a 32-byte (256-bit) random key, URL-safe base64 encoded
           raw_key = secrets.token_urlsafe(32)
           key_hash = hashlib.sha256(raw_key.encode()).hexdigest()

           with psycopg.connect(settings.database_url) as conn:
               with conn.cursor() as cur:
                   cur.execute(
                       "INSERT INTO api_keys (key_hash, name) VALUES (%s, %s) RETURNING id",
                       (key_hash, name),
                   )
                   key_id = cur.fetchone()[0]
               conn.commit()

           return raw_key


       if __name__ == "__main__":
           parser = argparse.ArgumentParser(description="Generate an API key")
           parser.add_argument(
               "--name",
               default="default",
               help="Human-readable name for the key (default: 'default')",
           )
           args = parser.parse_args()

           key = create_api_key(args.name)
           print(f"API Key (save this — it won't be shown again): {key}")
       ```

       Key design decisions:
       - Uses `secrets.token_urlsafe(32)` for cryptographically secure key generation (43 characters)
       - SHA-256 hash stored in DB — the raw key is never stored
       - Uses `psycopg.connect()` directly (not the pool) since this is a CLI script, not a web request
       - Prints the raw key to stdout so the user can copy it
       - The script imports `app.config` so it picks up DATABASE_URL from .env

    1. **app/dependencies.py** — Create FastAPI dependency for API key authentication:

       ```python
       """FastAPI dependencies for authentication and common parameters."""
       import hashlib

       from fastapi import Depends, HTTPException, Security
       from fastapi.security import APIKeyHeader

       from app.db.session import get_db

       # FastAPI security scheme — adds "Authorize" button in Swagger UI
       api_key_header = APIKeyHeader(name="X-API-Key", auto_error=False)


       def require_api_key(api_key: str | None = Security(api_key_header)) -> str:
           """Validate API key from X-API-Key header.

           Looks up SHA-256 hash of the provided key in the api_keys table.
           Returns the key name on success for logging/auditing.

           Raises:
               HTTPException 401: Missing or invalid API key
           """
           if api_key is None:
               raise HTTPException(
                   status_code=401,
                   detail="Missing API key. Include X-API-Key header.",
               )

           key_hash = hashlib.sha256(api_key.encode()).hexdigest()

           with get_db() as conn:
               with conn.cursor() as cur:
                   cur.execute(
                       "SELECT name FROM api_keys WHERE key_hash = %s AND active = true",
                       (key_hash,),
                   )
                   row = cur.fetchone()

           if row is None:
               raise HTTPException(
                   status_code=401,
                   detail="Invalid API key.",
               )

           return row[0]  # Return key name for logging
       ```

       Key design decisions:
       - Uses `APIKeyHeader` from `fastapi.security` — this adds the "Authorize" button in Swagger UI automatically
       - `auto_error=False` so we can provide custom error messages (not FastAPI's default 403)
       - SHA-256 hash comparison — constant-time at the DB level via index lookup
       - Returns key name (not key hash) so endpoints can log which key was used
       - Uses 401 (not 403) — 401 means "not authenticated", 403 means "authenticated but not authorized". For API keys, missing/invalid = not authenticated.
